VideLibri - Manages your lend books

VideLibri is a locally running extension of the usual library web catalogues. Its main function is to automatically extend the lend books and warn the user of an ending loan period, but it also has several other features.

It works by parsing the web page of the library, so it does not support all libraries.
In fact, I have only tested it with 5 libraries, and all of them are in a local region of Germany (read the German page for more information) and I have not translated the program into English, so far.

But it is not difficult to extend it with new libraries due to the simple templates it used. Especially if your library uses a system of libero, aleph or sisis, because in these cases you only have to change the server URL.
The template language is described on my homepage, and the currently existing templates are stored in the directory data/libraries/templates. In case you write your own template, you can test it with a single page online or with the example programs in directory components/pascal/data/example of the VideLibri source.

If you want to use/test it for your library, you can write me a mail. (I can also add the corresponding library, iff you can send me a library account with password)

Downloads

On the sourceforge download page you can download the Windows and Linux version of VideLibri. (Attention: Currently only a German version is available; but you can write me a mail, if you need a translation. When I uploaded VideLibri here, I did not expect that anyone outside of Germany would download it -- although many did. )

The source is stored in a mercurial repository.
Since the program is written in FreePascal/Lazarus, it should be very easy to compile it. Just open the bookWatch.lpi in Lazarus and select your platform/OS in the project settings. You can compile and run it on Windows/Linux/Mac with the gtk/qt/win32/cocoa interfaces (yes, all of them. Lazarus supports a lot of interfaces).




Spin-offs

Using the framework written for VideLibri it is trivial to convert other websites in local running programs. Since the share 90% of the source with VideLibri, I keep them in the same public repository:

Xidel

Xidel is just the VideLibri core with anything about libraries and books removed, and can be used to extract data from arbitrary webpages.
See more on the Xidel page.

SourceForgeResponder

This is a daemon that will convert every mail sent to localhost to a reply on the sourceforge page.
It is very easy to use:
  1. You start the daemon with your sourceforge username/password given as command line parameters
  2. If you get a sourceforge notification per mail, you send your reply to localhost (you have to cite the complete sourceforge mail, so the daemon knows where to post the reply)
  3. The daemon will automatically post it (but since it is currently single threaded, you have to wait 5 seconds before it ca post another reply )
Remark: You will not get a warning, if the daemon is unable to post a reply and the message will be silently lost. However, this does not really matter, because you will get a notification mail by sourceforge about every completed posting and your mail program should keep a copy of every send message (the daemon also prints every message to stdout).
You can read more details in the source.
